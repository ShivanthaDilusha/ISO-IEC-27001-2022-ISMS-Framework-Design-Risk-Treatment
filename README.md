# ISO-IEC-27001-2022-ISMS-Framework-Design-and-Risk-Treatment
Designed and developed a comprehensive ISMS core documentation pack for a simulated IT service provider, DSM IT (Pvt) Ltd in strict alignment with the ISO/IEC 27001:2022 standard. This project simulated real-world consulting to transition an organization from ad-hoc security practices to a formalized, audit-ready compliance posture.

Key Deliverables & Methodologies:

*Gap Analysis: Conducted a clause-by-clause assessment (Clauses 4–10) and Annex A controls analysis to map existing organizational practices against the ISO standard, identifying non-conformities and operational vulnerabilities.

*ISMS Scope Definition: Defined physical, logical, and organizational boundaries, aligning information security objectives with corporate risk tolerance and Sri Lanka’s Personal Data Protection Act (PDPA).

*Core Policy Formulation: Drafted the overarching corporate Information Security Policy, establishing roles, responsibilities (RACI), and top management commitments to continuous security improvement.

*Statement of Applicability (SoA): Selected, justified, and mapped key Annex A controls (including Access Control A.9, OpSec A.12, and Business Continuity A.17) against identified compliance demands.

*Risk Assessment & Treatment Plan: Constructed an initial enterprise Risk Register using a Likelihood × Impact methodology to prioritize high-risk vectors (e.g., unauthorized access, ransomware) and defined mitigation, transfer, and avoidance strategies with clear implementation timelines.
